# English LinkedIn Style Guide

Write polite, natural English LinkedIn posts about software projects. Default to the voice of someone trying or reviewing a repository rather than its creator.

## Voice

- Sound like a developer sharing useful notes, not a product advertisement.
- Keep the tone professional, humble, clear, and warm.
- Prefer concrete technical observations over grand claims.
- Use common terms such as repository, API, frontend, backend, workflow, deployment, and database naturally.
- Avoid excessive emojis, clickbait, hype, and corporate filler.
- Use a slightly personal rhythm instead of a generic essay.

## Point of View

### Reviewer/User POV

Use this by default for public repositories:

- "I spent some time exploring this repo..."
- "I tried this project to understand..."
- "What I found interesting was..."
- "One thing I learned from this repo..."

Do not imply ownership with phrases such as:

- "I built..."
- "I created..."
- "My project..."
- "A feature I developed..."

### Creator POV

Use this only when the user says the project is theirs:

- "I built..."
- "In this project, I tried to..."
- "One design decision I made..."
- "Next, I want to improve..."

## Recommended Post Shape

1. Open with one or two natural lines about the project.
2. Explain what the project appears to solve.
3. Mention its main features and stack.
4. Summarize the input-to-output workflow.
5. Share one concrete observation or learning.
6. Include the exact source URL when provided.
7. Close with a light invitation for discussion.

## Copyable Caption Format

Put the entire caption inside one fenced `text` block:

````markdown
## LinkedIn Status
```text
[Caption paragraphs]

Repo I tried:
[URL]

[Hashtags]
```
````

Do not split the caption into several blocks. Avoid bullets unless the user requests a list-style post.

## Length

- Short: 80 to 140 words.
- Medium: 150 to 260 words by default.
- Personal: similar to medium, with one clear learning reflection.

## Hashtags

Use 3 to 6 relevant tags. Select only tags supported by the project, such as:

```text
#SoftwareEngineering #WebDevelopment #OpenSource #MachineLearning #ReactJS #Python #BuildInPublic #LearningInPublic
```

## Avoid

- "This project is revolutionary..."
- "The best app for..."
- "Extremely powerful and amazing..."
- "Built with cutting-edge technology..."
- "In today's rapidly evolving digital era..."
- Claims that are larger than the repository evidence.

## Final Check

- Include at least one concrete observation from the source.
- Keep paragraphs between one and three sentences.
- Vary paragraph openings.
- Use measured language such as "interesting", "useful", "appears to", or "I still want to understand".
- Verify that the exact project URL is inside the caption.
